## 二、数据质量检查与清洗

### 2.1 发现的质量问题

通过扫描全部 3420 个标注文件，发现以下问题：

**错别字问题**：
- "内测" 应为 "内侧"（测 vs 侧）
- "肌建" 应为 "肌腱"（建 vs 腱）

**格式不一致**：
- `N-内测半月板` 多了连字符，应统一为 `N内侧半月板`

**缺少疾病前缀**：
- `内测半月板`、`腘窝囊肿` 等标签缺少疾病类别前缀

**占位符标签**：
- `滑膜那种` 应为 `滑膜囊肿-滑膜囊肿`

**疑似错误（需人工确认）**：
- `SPA-髌下深囊炎`（保留并标记）

**孤立文件**：
- 1 个 JSON 无对应图片（`正常/DEMO/内侧副韧带.json`）
- 35 张图片无对应 JSON 标注

## 三、数据集拆分

### 3.1 拆分策略

- **患者级拆分**：同一患者的所有图片只出现在同一集合，防止数据泄漏
- **分层抽样**：每个疾病类别内独立拆分，保证各集合疾病分布一致
- **比例**：train 70% / val 15% / test 15%（按患者数）
- **随机种子**：42（可复现）

## 四、数据格式转换

将清洗拆分后的数据转换为三种模型框架所需格式。

### 4.1 COCO 格式（用于 YOLO 路线）

| 集合 | 图片数 | 标注区域数 |
|------|--------|-----------|
| Train | 2349 | 5847 |
| Val | 514 | 1303 |
| Test | 556 | 1375 |

- 84 个区域类别统一编号（category_id: 1-84）
- 多边形坐标转 COCO segmentation 格式

### 4.2 YOLO-seg 格式

- 从 COCO 转为 YOLO-seg 标签（归一化多边形坐标）
- 每个图片一个 `.txt` 标签文件
- 自动生成训练配置 `configs/yolo_seg.yaml`

| 集合 | 标签文件数 |
|------|-----------|
| Train | 2349 |
| Val | 514 |
| Test | 556 |

### 4.3 nnU-Net 格式

- 多边形 → 像素级 PNG 掩码（84 类整数标签值）
- train + val 合并为 Tr（nnU-Net 自行交叉验证），test 作为 Ts

| 集合 | 样本数 |
|------|--------|
| Training (Tr) | 2863 |
| Testing (Ts) | 556 |

### 4.4 MedSAM 格式

- 每个标注对象 → 独立二值掩码 PNG + bbox 坐标
- 总共 8525 个 mask-bbox 对

| 集合 | 图片数 | mask-bbox 对数 |

| Train | 2345 | 5847 |

## 五、模型训练

### 5.1 训练环境

| 项目 | 配置 |
|------|------|
| GPU | NVIDIA GeForce RTX 3080 (10GB VRAM) |
| PyTorch | 2.6.0+cu124 |
| Python | 3.13.7 |

### 5.2 方案 A — YOLO11-seg 训练

- **模型**：YOLO11m-seg（22.4M 参数，113.9 GFLOPs）
- **预训练权重**：yolo11m-seg.pt（ImageNet + COCO 预训练）
- **训练配置**：
  - 输入尺寸：640×640（初始设为 1280，因 GPU 显存占满导致系统卡死，调整为 640）
  - Batch size：4（受 10GB 显存限制）
  - Epochs：300（early stopping patience=50）
  - 优化器：MuSGD(lr=0.01, momentum=0.9)，自动选择
  - 数据增强：HSV 扰动、水平翻转、Mosaic、缩放
  - 分类损失权重：0.5
- **输出**：84 类区域实例分割 + 基于前缀投票的 7 类疾病诊断
- **状态**：已完成
- **实际训练**：168 epochs（Epoch 118 最佳，之后早停）
- **训练耗时**：约 5.2 小时
- **GPU 显存占用**：~2.8G / 10G
