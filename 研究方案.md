### 1.2 标注格式

标注使用 ISAT（Image Segmentation Annotation Tool）工具生成，格式为 JSON，包含多边形轮廓标注。每个标注文件与同名图片对应，结构如下：

```json
{
    "info": {
        "description": "ISAT",
        "name": "image.jpg",
        "width": 1280,
        "height": 960,
        "depth": 3
    },
    "objects": [
        {
            "category": "RA滑膜血流",
            "group": 1,
            "segmentation": [[x1, y1], [x2, y2], ...],
            "area": 12345.0,
            "layer": 1,
            "bbox": [x_min, y_min, x_max, y_max],
            "iscrowd": false,
            "note": ""
        }
    ]
}
```

## 二、当前前沿方法与主流工具

### 2.1 实例分割模型（检测异常区域轮廓）

#### A. U-Net 及变体（仍然是主流）

- **nnU-Net** 是语义分割的金标准。MICCAI 2024 论文（arXiv:2404.09556）严格验证了在 nnU-Net 框架内正确配置的 CNN U-Net 模型在大多数数据集上优于 Transformer 和 Mamba 方案。
- **nnU-Net ResEnc M** 在 9GB 显存上训练 11 小时即可达到 87.91% Dice，优于 MONAI Auto3DSeg SegResNet（87.77% Dice / 240 GPU 小时 / 320GB 显存）。
- 专用变体：SK U-Net（选择性卷积核，乳腺超声 Dice 0.78）、MF U-Net（多尺度融合）、实时 U-Net（乳腺超声引导干预 Dice 90.5%）。

#### B. SAM / MedSAM / MedSAM2（基础模型）

- **MedSAM**（Nature Communications 2024）：在 157 万图像-掩码对上训练，覆盖 10 种模态（含超声）。比原版 SAM 提升 22.51% Dice。
- **LiteMedSAM**（2024.01）：比 MedSAM 快 10 倍，通过轻量编码器蒸馏实现。
- **MedSAM2**（2025.04）：新增 3D/视频支持（含实时超声），使用 SAM2 的 Hiera 骨干网络 + memory-attention 处理时序上下文。
- **ClickSAM**（SPIE 2024）：专门针对超声微调，乳腺超声 IoU 0.916 vs MedSAM 的 0.863。

#### C. YOLO 实例分割

- **YOLOv5-seg 甲状腺结节分割**（arXiv:2512.00639）：Dice 91%，mAP 0.87，适合临床实时部署。
- **YOLO11**（2024）：实例分割用于肿瘤边界描绘，实时处理。
- **YOLO26**（2025.09）：原生统一检测、实例分割、分类、姿态估计和 OBB，针对嵌入式/低功耗优化。

#### D. Mask R-CNN（基于 Detectron2）

- 仍广泛用于医学图像实例分割。MaskAppendix（2024，PMC）在 CT 分割上达到 87.17% DSC，优于 U-Net 和 DenseNet。
- 与你的数据高度匹配：Mask R-CNN 从多边形标注生成逐实例分割掩码，与 ISAT 标注格式完全一致。

#### E. Vision Transformers

- **SwinUNETR**（MONAI）：在 3D 分割上有竞争力，但直接对比中 nnU-Net 在 6 个数据集中赢了 5 个。
- Transformer + CNN 混合架构正成为联合分割+分类多任务学习的首选。

### 2.2 分类模型

| 方案 | 说明 |
|---|---|
| ResNet-50 / ResNet-101 | 超声图像分类基线 |
| EfficientNet / ConvNeXt | 高效分类架构 |
| Vision Transformers (ViT) | 可做对比学习自监督预训练 |

### 2.3 模型方案对比

| 框架 | 分割类型 | 分类能力 | 生产部署 | 医学影像专注度 | GPU 效率 |
|---|---|---|---|---|---|

## 三、关键开源框架与工具详情

### 3.1 nnU-Net v2（推荐用于最高分割精度）

- **GitHub**：https://github.com/MIC-DKFZ/nnUNet （6k+ stars）
- **优势**：全自动配置流水线，自动确定预处理、架构、训练参数。开箱即用的 SOTA。原生支持 2D（适合本数据）。GPU 极其高效（~11 小时，9GB 显存）。
- **局限**：仅语义分割（非实例分割）。无内置分类头。部署工具有限。
- **安装**：`git clone https://github.com/MIC-DKFZ/nnUNet.git && pip install -e .`
- **环境配置**：需设置 `nnUNet_raw`、`nnUNet_preprocessed`、`nnUNet_results` 目录的环境变量。
- **2D 训练**：`nnUNetv2_plan_and_preprocess -d DATASET_ID` → `nnUNetv2_train DATASET_ID 2d FOLD`
- **教程**：PYCAD Guide（2024）、Towards AI Guide（2024）

### 3.2 MONAI（推荐用于生产流水线）

- **GitHub**：https://github.com/Project-MONAI/MONAI （6k+ stars）
- **优势**：完整医学影像生态系统。DICOM/PACS 集成。联邦学习支持。Model bundles 部署。支持 SwinUNETR、SegResNet 等多种架构。
- **MONAI Label**：https://github.com/Project-MONAI/MONAILabel （600+ stars）—— AI 辅助标注，与 3D Slicer / OHIF 集成。发表于 Medical Image Analysis（2024）。
- **Auto3DSeg**：自动化分割流水线（计算效率不及 nnU-Net）。
- **MONet-FL**（MICCAI 2025）：将 nnU-Net 的分割优势与 MONAI 的生产生态桥接。

### 3.3 Ultralytics YOLO（推荐用于实例分割 + 实时推理）

- **GitHub**：https://github.com/ultralytics/ultralytics （40k+ stars）
- **优势**：原生实例分割。实时推理。内置 COCO 格式支持。简单训练 API。直接支持多边形标注。可导出 ONNX/TensorRT 部署。YOLO26 原生统一分割+分类。
- **适配本数据**：可直接消费 YOLO 分割格式的多边形标注（每行归一化坐标）。
- **训练命令**：`yolo segment train data=knee.yaml model=yolo11n-seg.pt epochs=100 imgsz=640`
- **文档**：https://docs.ultralytics.com/tasks/segment/

### 3.4 Detectron2（推荐用于 Mask R-CNN 实例分割）

- **GitHub**：https://github.com/facebookresearch/detectron2 （30k+ stars）
- **优势**：SOTA 的 Mask R-CNN、Cascade Mask R-CNN、PointRend。COCO 格式原生支持。TorchScript/ONNX 导出。模块化架构。多 GPU 分布式训练。
- **适配本数据**：注册 COCO 格式数据集，微调预训练 Mask R-CNN。适合同时需要检测框和实例掩码的场景。

### 3.5 MMSegmentation（语义分割备选）

- **GitHub**：https://github.com/open-mmlab/mmsegmentation （8k+ stars）
- **优势**：统一基准测试平台，支持 DeepLabV3+、PSPNet、U-Net、SegNeXt、PIDNet。配置驱动。
- **局限**：主要是语义分割。开发活跃度不如其他框架。医学影像专用工具不如 MONAI/nnU-Net。

## 五、多边形标注训练最佳实践

基于 Supervisely 指南和 Label Your Data（2026）的建议：

1. **标准化标注流程**：建立清晰的顶点密度、重叠结构处理和边缘情况规范。本数据集已通过疾病前缀命名规范做了良好实践（N-、RA-、GA- 等）。

2. **平衡顶点密度**：更高密度捕获更多细节但增加文件大小和训练复杂性。超声图像通常边界模糊，适合中等密度。本数据标注使用密集多边形（每对象约 5-200 个顶点），是合适的。

3. **处理重叠/遮挡结构**：ISAT 标注支持 `group` 和 `layer` 字段管理遮挡顺序——需一致使用。

4. **标注者一致性**：医学影像应以 IoU >= 0.75-0.85 为目标（因超声固有的边界模糊性）。需高年资专家裁定分歧。

5. **使用 AI 辅助标注**：ISAT 的 SAM 集成可减少 40-60% 标注时间。可在 ISAT 中配置 MedSAM 作为后端模型，获得超声专用辅助。

6. **质量控制**：定期抽检标注。使用 ISAT 的掩码可视化功能验证分割质量。

7. **多边形标注的训练优势**：使用多边形标注训练的模型比基于边界框的弱监督方法分割精度高约 18%。U-Net 类训练时转为二值掩码，YOLO/Mask R-CNN 可保留多边形格式。

## 六、多任务学习（分割 + 分类联合训练）

### 6.1 架构范式

主流方法是**共享编码器 + 任务专用解码头**：

```
输入图像 → 共享编码器 (ResNet / EfficientNet / ViT)
                    |
            +-------+--------+
            |                 |
    分割解码器              分类头
    (U-Net 解码路径)       (全局平均池化 + FC)
            |                 |
    像素级掩码             疾病类别
```

### 6.2 关键近期论文

- **Lu et al.（Frontiers in Oncology，2025.04）**："Automatic joint segmentation and classification of breast ultrasound images via multi-task learning with object contextual attention." —— 与本项目直接相关，目标为超声多任务学习。

- **Multi-Scale Feature Enhancement MTL（arXiv/ScienceDirect，2024-2025）**：基于 UNet 的多任务学习，使用新型 **ResFormer block**（CNN + Transformer 混合）编码器和 **Dilated Feature Enhancement (DFE)** 分割解码模块。双任务同时达到优越性能。

- **Multi-Task DL for Cancer Pathologies（MDPI，2025）**：使用 VGG16/MobileNetV2 骨干的 UNet，实现四种癌症类型的同步分类与分割。

- **BrainTumNet（Frontiers in Oncology，2025.05）**：自适应掩码 Transformer 用于多任务脑肿瘤分析。

### 6.3 本数据集的多任务学习策略

本数据集天然支持多任务学习：

- **分割标签**：多边形标注的解剖结构
- **分类标签**：文件夹结构定义 7 种疾病类别
- **标签前缀编码疾病**：N-（正常）、RA-（类风湿性关节炎）、GA-（痛风性关节炎）等。分割标签本身携带分类信息。

**推荐多任务架构选项：**

**选项 A —— YOLO26-seg**：原生统一实例分割和分类。每个检测到的结构同时获得掩码和类别标签。疾病前缀类别（如 "RA积液" vs "N积液"）直接编码解剖结构和病理状态。

**选项 B —— 自定义 MTL 共享编码器**：
- 编码器：ResNet-50 或 EfficientNet-B4（ImageNet 预训练）
- 分割解码器：U-Net 风格上采样路径
- 分类头：全局平均池化 + FC 层（7 类疾病预测）
- 损失函数：`L_total = λ_seg × L_dice_ce + λ_cls × L_cross_entropy`
- 框架：PyTorch + MONAI 组件

**选项 C —— 两阶段流水线**：
- 阶段 1：实例分割（YOLO-seg 或 Mask R-CNN）检测并分割所有结构
- 阶段 2：基于检测到的结构及其形态学特征进行分类
- 这更接近临床推理过程

## 七、SAM / MedSAM 医学超声微调

### 7.1 关键资源

| 资源 | URL | 用途 |
|---|---|---|
| **MedSAM**（官方） | https://github.com/bowang-lab/MedSAM | 预训练权重、快速入门 notebook |
| **Mazurowski Lab finetune-SAM** | https://github.com/mazurowski-lab/finetune-SAM | 最佳逐步微调框架，支持 adapter |

| **CVPR24 DAFT** | https://github.com/automl/CVPR24-MedSAM-on-Laptop | 数据感知微调脚本，模态专用 |
| **MedSAM2**（最新） | LearnOpenCV 详解 | 3D/视频/实时超声支持 |
| **SAM2 微调教程** | DataCamp Tutorial | 逐步 SAM2 医学数据微调 |

### 7.2 推荐微调策略

基于 Mazurowski Lab 的研究成果：

1. **从 MedSAM 预训练权重出发**（`medsam_vit_b.pth`）——已在包含超声在内的多模态数据上训练。
2. **使用 [0, 1] 归一化**（不用 ImageNet 的 mean/std），适配 MedSAM 权重。
3. **参数高效微调（PEFT）**：使用 LoRA adapter 微调编码器和解码器：
   - `args.if_mask_decoder_adapter = True`
   - `args.if_update_encoder = True`
   - `args.if_encoder_adapter = True`
4. **数据准备**：将 ISAT 多边形转为二值掩码（每类一个）。创建 2D PNG 图像 + 掩码对。患者级 80/20 拆分（防数据泄漏）。
5. **提示策略**：使用 ISAT 标注中的 bbox 作为边界框提示，或使用多边形质心作为点击提示。
6. **超声专用注意事项**：SAM-UNet 研究表明轻量架构（SAM-UNet-34）在超声上优于重型架构（SAM-UNet-50），暗示超声可能受益于更轻量的修改。

### 7.3 在 ISAT 中使用 MedSAM 辅助标注

ISAT 原生支持 MedSAM 作为后端模型。可通过配置 ISAT 的模型设置来使用 MedSAM 加速剩余未标注图像的标注工作。

## 八、膝关节超声 AI 分析相关研究

### 8.1 直接相关论文

- **Segmentation and Classification of Knee Joint US Images via Deep Learning（ScienceDirect）**：最直接相关的论文。提出 snake 预处理 + 膨胀卷积 + 多通道学习的分割框架，以及 ResNet 分类 + 图嵌入的 6 类膝关节超声分类（正常、有/无滑膜增厚的积液、囊肿、肿瘤、类风湿性关节炎）。数据集仅 600 张超声图。

- **Survey on DL for Knee Joint Ultrasound（Medical AI，2025）**：综合综述，涵盖 Mask R-CNN 膝关节软骨分割、数据稀缺挑战和未来方向。

- **AI in Musculoskeletal US（La Radiologia Medica，2024）**：16 项研究的系统综述——69% 使用深度学习，69% 目标为诊断和/或分割。无一项研究报告了外部验证，凸显了该领域的重大缺口。

- **Meniscus Segmentation with YOLO（MDPI，2025）**：YOLO 集成方法用于膝关节 MRI 半月板分割。

- **OA Research with AI（Nature Bone Research，2025）**：AI 用于早期 OA 筛查、自动分级和形态学分析。

### 8.2 本数据集在领域中的定位

本数据集的 730 患者、6,866 张图、3,420 多边形标注、7 种疾病类别的规模，远超大多数已发表的膝关节超声 AI 研究。系统综述指出数据集稀缺和缺乏外部验证是该领域两大瓶颈。本数据集标注方案覆盖约 30+ 解剖结构并附带疾病专用标签，比大多数已发表工作更加详尽。

## 九、推荐实施方案（三条路线）

### 方案 A：YOLO 实例分割 + 疾病分类（推荐首选，最快落地）

```
ISAT标注 → COCO格式 → YOLO-seg格式 → YOLO11-seg训练
                                          ↓
                               每个区域输出：类别 + 多边形掩码
                                          ↓
                           后处理：根据检测到的类别前缀(RA/OA/GA等)
                                   →  聚合推断疾病类型
```

**核心步骤：**

1. **格式转换**：使用 ISAT 内置 `isat_to_coco()` 转为 COCO，再用 `ultralytics.data.converter.convert_coco()` 转为 YOLO-seg 格式
2. **统一标签**：清洗 86 个类别名中的不一致项（如 "N内测副韧带" vs "N内侧副韧带"）
3. **患者级拆分**：70/15/15 按患者拆分 train/val/test，防止数据泄漏
4. **训练**：`yolo segment train data=knee.yaml model=yolo11m-seg.pt epochs=300 imgsz=1280`
5. **推理**：检测到的区域类别前缀 → 聚合判断疾病

**优势**：一个模型同时完成分割和分类，训练部署简单，实时推理。

---

### 方案 B：nnU-Net 语义分割 + 独立分类器（最高分割精度）

```
ISAT标注 → VOC掩码(PNG) → nnU-Net格式
                              ↓
                     nnU-Net 2D自动训练
                              ↓
                     像素级分割图（30+解剖结构）

原始图片 → ResNet-50/EfficientNet → 7类疾病分类
```

**核心步骤：**

1. **格式转换**：ISAT → VOC 掩码 PNG，再组织为 nnU-Net 目录结构（`imagesTr/`、`labelsTr/`）
2. **nnU-Net 训练**：`nnUNetv2_plan_and_preprocess -d 001` → `nnUNetv2_train 001 2d all`
3. **分类器训练**：单独训练 EfficientNet-B4，输入原图，输出 7 类疾病
4. **推理**：分割模型标注区域 + 分类模型判断疾病

**优势**：分割精度最高（nnU-Net 在 MICCAI 2024 实验中优于 Transformer 方案），GPU 极其高效。

---

### 方案 C：MedSAM 微调（基础模型路线，前沿学术价值高）

```
ISAT标注 → 二值掩码 + bbox提示
                ↓
    MedSAM预训练权重 + LoRA微调（mazurowski-lab/finetune-SAM框架）
                ↓
    提示式分割模型（给bbox即可分割任意结构）
                ↓
    + 分类头（从编码器特征做7类分类）
```

**核心步骤：**

1. 下载 MedSAM 预训练权重 `medsam_vit_b.pth`
2. 使用 mazurowski-lab/finetune-SAM 框架（https://github.com/mazurowski-lab/finetune-SAM）
3. 将 ISAT 标注转为 binary mask + bbox prompt 对
4. 用 LoRA adapter 微调 encoder + decoder
5. 在 encoder 输出上加分类头做多任务训练

**优势**：学术前沿，论文发表价值最高。MedSAM2 还支持超声视频实时分析。

## 十、关键实施要点

### 10.1 数据预处理

- **患者级拆分**：同一患者的所有图片只能出现在同一集合中，防止数据泄漏。推荐 70/15/15 或 80/10/10。
- **标签清洗**：统一类似 "内测" → "内侧" 等不一致命名。
- **类别不平衡处理**：滑膜囊肿仅 36 个标注 vs RA 1,006 个，需使用 Focal Loss / 过采样 / 类别加权损失函数。
- **超声有效区域裁剪**：去除机器 UI 边框和文字叠加区域。

### 10.2 数据增强（超声专用）

- 水平翻转
- 随机亮度/对比度调整
- 弹性形变
- 高斯噪声
- 随机裁剪缩放

### 10.3 评估指标

- **分割**：Dice Score、IoU (Jaccard)、逐类别 mAP
- **分类**：Accuracy、F1-Score、Confusion Matrix、AUC-ROC
- **临床**：灵敏度（Sensitivity）、特异度（Specificity）

### 10.4 格式转换速查表

| 目标框架 | 转换路径 |
|---|---|
| YOLO-seg | ISAT → `isat_to_coco()` → `convert_coco(use_segments=True)` |
| Detectron2 | ISAT → `isat_to_coco()` → 直接注册 COCO 数据集 |
| nnU-Net | ISAT → `isat_to_voc()` → 重组目录为 nnU-Net 格式 |
| MedSAM | ISAT → 提取 bbox + 生成 binary mask PNG |

## 十一、推荐实施顺序

1. **第一步**：数据清洗 + 标签统一 + 患者级拆分
2. **第二步**：先跑 YOLO11-seg（方案 A），快速获得 baseline 结果
3. **第三步**：跑 nnU-Net（方案 B）对比分割精度
4. **第四步**：如需发论文，微调 MedSAM（方案 C）做创新点
5. **第五步**：最优模型集成 + 部署（MONAI / TorchServe / ONNX）
