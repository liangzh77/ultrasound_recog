# 膝关节超声数据集说明

## 一、数据来源与结构

原始数据位于 `data/膝关节已标注/`，按疾病类别分为 7 个子目录，每个子目录下按患者组织文件夹。

```
data/膝关节已标注/
├── 正常/
│   ├── 患者A/
│   │   ├── image1.png
│   │   ├── image1.json   ← labelme 格式标注
│   │   └── ...
│   └── ...
├── 类风湿性关节炎/
├── 骨性关节炎/
├── 痛风性关节炎/
├── 脊柱关节炎/
├── 损伤/
└── 滑膜囊肿/
```

标注文件为 **labelme JSON 格式**，包含多边形（polygon）分割区域，每个区域带有解剖结构标签。

## 三、标注质量与清洗

### 3.1 发现的问题

原始数据存在以下标注质量问题：

| 问题类型 | 示例 |
|----------|------|
| 错别字 | `内测` → `内侧`，`肌建` → `肌腱` |
| 格式不一致 | `N-内测半月板` → `N内侧半月板` |
| 缺疾病前缀 | `内测半月板` → `N内侧半月板` |
| 占位符标签 | `滑膜那种` → `滑膜囊肿-滑膜囊肿` |

共清洗 **155 个 JSON 文件**，修正 **197 处**标签错误。清洗后数据输出至 `data/cleaned/`，原始数据不变。

### 3.2 缺标注图片

35 张图片有图无 JSON 标注，分布为：

| 疾病 | 缺标注图片数 |
|------|-------------|
| 正常 | 10 |
| 类风湿性关节炎 | 14 |
| 痛风性关节炎 | 9 |
| 骨性关节炎 | 2 |

## 四、数据集拆分

### 4.1 拆分策略

- **患者级拆分**：同一患者的所有图片只出现在同一子集，避免数据泄漏
- **分层抽样**：每个疾病类别内独立拆分，保证各子集疾病分布一致
- **比例**：Train 70% / Val 15% / Test 15%（按患者数计算）
- **随机种子**：42（可复现）

## 五、标注区域类别体系

每张有标注图片包含若干解剖结构的多边形分割，标签格式为 `{疾病前缀}{解剖结构名}`，例如：

- `N内侧半月板`（正常·内侧半月板）
- `RA滑膜`（类风湿·滑膜）
- `OA骨赘`（骨性关节炎·骨赘）

84 个标注类别涵盖韧带、肌腱、半月板、滑膜、囊肿、骨赘等膝关节解剖结构，各疾病关注的解剖区域有所不同。

## 六、主要挑战

| 挑战 | 说明 |
|------|------|
| 类别严重不平衡 | 滑膜囊肿仅 36 张有标注图片，为最大类的 1/28 |
| 小样本类别 | 滑膜囊肿（11 患者）、损伤（38 患者）样本极少 |
| 细粒度标注 | 84 个解剖区域类别，部分类别样本量极少（如髌下深囊） |
| 标注噪声 | 原始数据存在错别字、格式不一等问题，需清洗后使用 |
