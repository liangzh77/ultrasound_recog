## 二、数据质量检查与清洗

### 2.1 发现的质量问题

通过扫描全部 3420 个标注文件，发现以下问题：

**错别字问题**：
- "内测" 应为 "内侧"（测 vs 侧）
- "肌建" 应为 "肌腱"（建 vs 腱）

**格式不一致**：
- `N-内测半月板` 多了连字符，应统一为 `N内侧半月板`

**缺少疾病前缀**：
- `内测半月板`、`腘窝囊肿` 等标签缺少疾病类别前缀

**占位符标签**：
- `滑膜那种` 应为 `滑膜囊肿-滑膜囊肿`

**疑似错误（需人工确认）**：
- `SPA-髌下深囊炎`（保留并标记）

**孤立文件**：
- 1 个 JSON 无对应图片（`正常/DEMO/内侧副韧带.json`）
- 35 张图片无对应 JSON 标注

## 三、数据集拆分

### 3.1 拆分策略

- **患者级拆分**：同一患者的所有图片只出现在同一集合，防止数据泄漏
- **分层抽样**：每个疾病类别内独立拆分，保证各集合疾病分布一致
- **比例**：train 70% / val 15% / test 15%（按患者数）
- **随机种子**：42（可复现）

### 5.2 方案 A — YOLO11-seg 训练

- **模型**：YOLO11m-seg（22.4M 参数，113.9 GFLOPs）
- **预训练权重**：yolo11m-seg.pt（ImageNet + COCO 预训练）
- **训练配置**：
  - 输入尺寸：640×640（初始设为 1280，因 GPU 显存占满导致系统卡死，调整为 640）

  - Epochs：300（early stopping patience=50）
  - 优化器：MuSGD(lr=0.01, momentum=0.9)，自动选择
  - 数据增强：HSV 扰动、水平翻转、Mosaic、缩放

- **输出**：84 类区域实例分割 + 基于前缀投票的 7 类疾病诊断
- **状态**：已完成
- **实际训练**：168 epochs（Epoch 118 最佳，之后早停）

- **GPU 显存占用**：~2.8G / 10G

### 5.6 方案 C — MedSAM 微调

- **模型**：MedSAM（ViT-B，medsam_vit_b.pth 预训练）
- **任务**：bbox prompt 分割 + 7 类疾病分类（联合训练）
- **训练策略**：
  - 冻结 image encoder，只训练 prompt encoder + mask decoder + 分类头
  - 预计算 image embeddings 缓存到磁盘（避免重复编码，训练加速 ~150 倍）
- **训练配置**：
  - 输入尺寸：1024×1024（encoder）→ 256×256（mask 输出）
  - Batch size：8
  - Epochs：50
  - 优化器：AdamW（decoder lr=1e-5, classifier lr=1e-4）+ CosineAnnealingLR
  - 损失：Dice Loss + BCE（分割）+ CrossEntropy（分类），权重 1.0:0.5
- **状态**：已完成
- **GPU 显存占用**：encoder 冻结后显存可控
