# 膝关节超声 AI 识别系统 — 研究总结

---

## 一、研究概述

本研究基于 730 名患者、6866 张膝关节超声图像（其中 3420 张已标注），构建了覆盖 7 类疾病、84 类解剖区域的 AI 识别系统。采用三条并行技术路线，在同一数据集、同一拆分、同一评估标准下进行系统性对比。

### 1.1 三条技术路线

| 方案 | 分割方法 | 分类方法 | 范式定位 |
|------|----------|----------|----------|
| 方案 A | YOLO11m-seg（实例分割） | EfficientNet-B0（独立分类器） | 工业界主流，速度优先 |
| 方案 B | nnU-Net v2（语义分割） | EfficientNet-B0（独立分类器） | 医学影像金标准 |
| 方案 C | MedSAM（bbox prompt 分割） | 联合训练分类头 | 基础模型微调，前沿方向 |

### 1.2 核心指标汇总

**分割指标**：

| 方案 | 指标 | 数值 | 评估集 |
|------|------|------|--------|
| MedSAM | Mean Dice | **0.8331** | val (1303 samples) |
| YOLO11-seg v2 | Mask mAP50 | 0.677 | val (514 images) |
| YOLO11-seg v2 | Mask mAP50-95 | 0.381 | val (514 images) |
| nnU-Net | Mean Dice | 0.497 | fold_0 val (573 images) |

**分类指标**：

| 方案 | Accuracy | F1 Weighted | F1 Macro | 评估集 |
|------|----------|-------------|----------|--------|
| EfficientNet-B0 | **57.19%** | **0.550** | **0.402** | test (556 images) |
| MedSAM 联合分类 | 47.58% | 0.348 | 0.171 | val (1303 samples) |

## 二、研究亮点

### 2.1 数据集的稀缺性与标注深度

膝关节超声 AI 领域的公开研究远少于 X-ray、CT、MRI。本数据集具备以下特点：

- **7 类疾病覆盖**：正常、类风湿性关节炎、骨性关节炎、痛风性关节炎、脊柱关节炎、损伤、滑膜囊肿
- **84 类解剖区域标注**：多边形分割标注（非简单分类标签），标注粒度细
- **730 名患者、3420 张标注图片**：在超声领域属中等偏上规模
- **患者级数据拆分**：防止同一患者数据出现在训练集和测试集中，避免数据泄漏

类似规模和标注深度的膝关节超声公开 benchmark 极为稀少，数据集本身具有学术贡献价值。

### 2.2 三方案系统性对比（核心亮点）

在**同一数据集、同一拆分策略、同一评估标准**下，对三种代表性范式进行 apple-to-apple comparison：

| 维度 | YOLO11-seg | nnU-Net | MedSAM |
|------|------------|---------|--------|
| 分割范式 | 单阶段实例分割 | 自配置语义分割 | 基础模型 prompt 分割 |
| 分割类别 | 21 类解剖结构 | 22 类解剖结构 | 逐目标二值分割 |
| 分类方式 | 独立 EfficientNet-B0 | 独立 EfficientNet-B0 | 联合训练分类头 |
| 训练耗时 | ~5.4 小时 | ~21.5 小时 | ~3 小时 |
| 推理速度 | ~10ms/图 | 需命令行调用 | ~2-3s/图 |
| 分割最佳指标 | mAP50=0.677 | Dice=0.497 | Dice=0.833 |
| 分类最佳指标 | Acc=57.19%（共享） | Acc=57.19%（共享） | Acc=47.58% |

**主要发现**：
- **MedSAM 分割最优**（Dice 0.833），但依赖 bbox prompt 输入，且分类能力受限于冻结 encoder
- **YOLO 综合性价比最高**：推理速度最快（~10ms），分割精度良好，端到端流程最完整
- **nnU-Net 未达预期**：可能受限于仅训练 fold 0 及 10GB 显存对 patch size 的限制

这种多范式对比在医学超声领域的论文中不常见，对后续研究者选择技术路线有直接参考价值。

### 2.3 标签合并策略的关键发现

原始 84 类标签（疾病前缀 × 解剖结构）合并为 21 类（纯解剖结构）后，效果显著提升：

| 指标 | 合并前（84 类） | 合并后（21 类） | 提升幅度 |
|------|-----------------|-----------------|----------|
| YOLO Mask mAP50 | 0.467 | **0.677** | **+45%** |
| YOLO Mask mAP50-95 | 0.276 | **0.381** | **+38%** |
| YOLO Box mAP50-95 | 0.276 | **0.462** | **+67%** |
| nnU-Net Dice | 0.000（无法学习） | **0.675** | 从零到可用 |

**核心结论**：在类别数远超样本量支撑能力时，合并语义相近类别比增大模型参数更有效。nnU-Net 在 84 类时训练速度慢 60 倍（~80 min/epoch vs ~80s/epoch）且 Dice 持续为 0，合并后立即开始学习。

这是一个可复现、可推广到其他医学影像多类别分割任务的实用结论。

### 2.4 资源受限下的大模型微调策略

在消费级 GPU（RTX 3080, 10GB VRAM）上微调 MedSAM（ViT-B, 93M 参数）面临显存不足的挑战。本研究提出的优化方案：

1. **冻结 image encoder** + `torch.no_grad()`：显存从 ~18GB 降到 ~1GB
2. **预计算 image embeddings 缓存到磁盘**：5847 个样本共享 2345 张图片，一次性编码后以 float16 缓存（~5GB 磁盘空间）
3. 训练时直接加载 embedding（256×64×64），跳过 encoder 前向传播

**效果**：
- 训练速度提升 **~150 倍**（从 ~60s/iteration 到 ~2.5 it/s）
- 单 epoch 时间从 ~48 小时降至 ~5 分钟
- 分割 Dice 仍达 **0.833**，为所有方案最高

这一策略对资源受限环境下微调视觉基础模型（SAM、SAM2 等）具有实际参考意义。

### 2.5 完整可复现的工程流水线

12 个脚本覆盖从原始数据到模型部署的全流程：

```
01_clean_labels.py      → 标签清洗（155 文件，197 处修正）
02_split_dataset.py     → 患者级分层拆分
03_convert_coco.py      → COCO 格式转换
04_convert_yolo.py      → YOLO-seg 格式转换
05_convert_nnunet.py    → nnU-Net 格式转换
06_convert_medsam.py    → MedSAM 格式转换
07_train_yolo.py        → YOLO11-seg 训练
08_train_nnunet.sh      → nnU-Net 训练
09_train_classifier.py  → EfficientNet-B0 分类器训练
10_train_medsam.py      → MedSAM 微调
11_evaluate.py          → 统一评估
12_inference.py         → 统一推理 + ONNX 导出
```

## 三、客观局限性

| 局限 | 具体表现 | 可能的改进方向 |
|------|----------|----------------|
| 分类精度偏低 | 7 类疾病最佳仅 57.19% | 引入多模态特征（分割区域组合模式）、更强的数据增强 |
| 类别严重不平衡 | 滑膜囊肿 36 张 vs 类风湿 1006 张，小类 F1≈0 | 数据采集、合成数据增强、few-shot learning |
| 单中心数据 | 所有数据来自同一医院 | 多中心验证以评估泛化能力 |
| nnU-Net 仅 fold 0 | 未完成完整 5-fold 交叉验证 | 完成全部 5 折训练取平均 |
| MedSAM 分类受限 | 冻结 encoder 导致分类仅 47.58% | 尝试部分解冻 encoder 或 LoRA 微调 |
| 分割与分类指标不完全可比 | MedSAM 逐 bbox 二值分割 vs YOLO/nnU-Net 多类分割 | 设计统一的评估协议 |

## 四、论文发表建议

### 4.1 推荐选题角度

**标题方向**：「膝关节超声多病种 AI 辅助诊断：三种分割-分类范式的系统性对比研究」

**核心卖点**：
1. 三方案 apple-to-apple 对比（YOLO vs nnU-Net vs MedSAM）
2. 标签合并策略对分割性能的显著影响
3. 资源受限下基础模型微调的工程优化

### 4.2 论文结构建议

| 章节 | 重点内容 |
|------|----------|
| Introduction | 膝关节超声 AI 现状、多范式对比的必要性 |
| Related Work | YOLO-seg、nnU-Net、SAM/MedSAM 各自的发展 |
| Dataset | 数据集描述、标注规范、清洗流程、拆分策略 |
| Methods | 三条技术路线的实现细节、标签合并策略、MedSAM 优化方案 |
| Experiments | 统一评估指标、分割对比、分类对比、消融实验（84 vs 21 类） |
| Discussion | 各方案优劣分析、临床应用场景讨论、局限性 |
