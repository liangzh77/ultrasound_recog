# 膝关节超声 AI 识别系统 — 开发日志

## 一、项目概述

**目标**：基于已标注的膝关节超声图像数据，构建 AI 系统实现区域分割 + 疾病分类。

**技术路线**：三条并行方案
- 方案 A：YOLO11-seg（端到端实例分割）
- 方案 B：nnU-Net + EfficientNet-B4（语义分割 + 独立分类器）
- 方案 C：MedSAM 微调（基础模型 + 分类头联合训练）

### 2.3 发现的数据质量问题

通过遍历所有 JSON 标注文件，发现以下问题：

1. **错别字**："内测" 应为 "内侧"、"肌建" 应为 "肌腱"
2. **格式不一致**：`N-内测半月板` 多了连字符，应为 `N内侧半月板`
3. **缺少疾病前缀**：`内测半月板`、`腘窝囊肿` 缺少疾病类别前缀
4. **占位符标签**：`滑膜那种` 应为 `滑膜囊肿-滑膜囊肿`
5. **疑似错误**：`SPA-髌下深囊炎`（保留并标记，需人工确认）
6. **孤立文件**：1 个 JSON 无对应图片，35 张图片无对应 JSON

### 4.2 各模块实现详情

#### 4.2.1 标签映射模块 — `src/label_mapping.py`

定义了标签清洗的核心规则：

```python
LABEL_FIX_MAP = {
    "N内测副韧带": "N内侧副韧带",
    "N-内测半月板": "N内侧半月板",
    "SPA-股二头肌建": "SPA-股二头肌腱",
    "滑膜那种": None,  # 需结合疾病目录补全
}
```

同时定义了 7 类疾病映射、86 个区域类别到疾病的映射关系、以及从标签提取解剖结构的函数。

#### 4.2.2 标签清洗 — `scripts/01_clean_labels.py`

- 遍历 `data/膝关节已标注/` 下所有 JSON
- 应用修复规则（错别字、补前缀、占位符替换）
- 不覆盖原文件，输出到 `data/cleaned/`
- 生成清洗报告 `data/clean_report.json` 和类别映射 `data/category_mapping.json`
- 支持 `--dry-run` 模式

#### 4.2.3 数据集拆分 — `scripts/02_split_dataset.py`

- **患者级拆分**：同一患者的所有图片只出现在同一集合，防止数据泄漏
- **分层抽样**：每个疾病类别内独立拆分，保证各集合疾病分布一致
- **比例**：train 70% / val 15% / test 15%（按患者数）
- 输出 `data/splits/{train,val,test}.txt` + `split_stats.json`

#### 4.2.4 格式转换（scripts/03-06）

| 脚本 | 输入 | 输出 | 说明 |
|------|------|------|------|
| 03_convert_coco.py | 清洗后 JSON + split | `data/coco/{split}/annotations.json` + images/ | 标准 COCO 实例分割格式 |
| 04_convert_yolo.py | COCO annotations | `data/yolo_seg/images/{split}/` + `labels/{split}/` | YOLO-seg 归一化坐标格式 |
| 05_convert_nnunet.py | 清洗后 JSON + split | `data/nnunet/Dataset001_Knee/` | 像素级 PNG 掩码 |
| 06_convert_medsam.py | 清洗后 JSON + split | `data/medsam/{split}/` | 二值掩码 + bbox JSON |

#### 4.2.5 训练脚本（scripts/07-10）

**方案 A — YOLO11-seg**（`07_train_yolo.py`）：
- 模型：YOLO11m-seg 预训练权重
- 输入尺寸：640，epochs: 300，patience: 50
- 疾病分类：检测结果前缀投票机制

**方案 B — nnU-Net + 分类器**（`08_train_nnunet.sh` + `09_train_classifier.py`）：
- 分割：nnU-Net v2 自配置 5-fold 交叉验证
- 分类：EfficientNet-B4，WeightedRandomSampler 处理类别不平衡

**方案 C — MedSAM**（`10_train_medsam.py`）：
- MedSAM ViT-B + 7 类分类头联合模型
- 输入：1024×1024，bbox prompt
- 损失：`λ_seg × (DiceLoss + BCE) + λ_cls × CrossEntropy`
- 分割和分类使用不同学习率（1:10）

#### 4.2.6 评估模块 — `src/metrics.py` + `scripts/11_evaluate.py`

评估指标：
- **分割**：Dice Score、IoU、mAP@0.5、mAP@0.5:0.95
- **分类**：Accuracy、Weighted F1、Confusion Matrix、AUC-ROC
- **可视化**：混淆矩阵热力图、方案对比柱状图、掩码叠加图

#### 4.2.7 推理部署 — `scripts/12_inference.py`

- 统一接口：`--method yolo|nnunet|medsam --image <path>`
- YOLO：完整检测 + 掩码轮廓 + 疾病投票
- nnU-Net：EfficientNet 分类 + nnUNetv2_predict 命令
- 支持 `--export-onnx` 导出 YOLO ONNX 模型
- 输出 JSON 结果文件

### 8.2 方案 A — YOLO11-seg v2（合并标签 21 类）

- **状态**：已完成
- **模型**：YOLO11m-seg（与 v1 相同）
- **关键改动**：原始 84 类按解剖结构合并为 21 类（去除疾病前缀），与 nnU-Net 合并策略一致
- **数据集配置**：`configs/yolo_seg_merged.yaml`
- **配置**：imgsz=640, batch=4, epochs=300, patience=50
- **输出目录**：`runs/yolo_seg/knee_v2/`
- **实际训练**：179 epochs（Epoch 165 为最佳）
- **训练耗时**：约 5.4 小时
- **GPU 显存**：~2.8G / 10G（与 v1 相同）

#### 合并标签策略

与 nnU-Net 类别合并方案一致，去除疾病前缀（N、RA、OA、GA、SPA-、损伤-、滑膜囊肿-），仅保留解剖结构名称，从 84 类降至 21 类。配置文件 `configs/yolo_seg_merged.yaml` 定义了 21 个类别：滑膜、滑膜血流、积液、软骨、骨皮质、骨侵蚀、骨赘、股四头肌腱、髌韧带、内侧副韧带、外侧副韧带、内侧半月板、半膜肌腱、股二头肌腱、髂胫束、鹅足腱、囊肿、痛风石、结晶、髌前区、髌下深囊。

#### 与 v1 对比

| 指标 | v1（84 类） | v2（21 类） | 提升 |
|------|------------|------------|------|
| Mask mAP50 | 0.467 | **0.677** | +45% |

#### 分析

1. **标签合并效果显著**：类别从 84→21 后，mAP50 提升 45%，mAP50-95 提升 38%，与 nnU-Net 的经验一致
2. **cls_loss 大幅下降**：v1 最终 cls_loss=0.456 vs v2 最终 cls_loss=0.376，类别少更容易学习
3. **Box 指标提升更大**：Box mAP50-95 从 0.276→0.462（+67%），说明定位精度也显著改善
4. **目前最佳分割方案**：YOLO v2 的 mAP50=0.677 是所有方案中实例分割的最佳结果
5. **疾病分类需额外步骤**：合并标签后失去了疾病前缀信息，需结合分类器或分析区域组合来推断疾病类型

### 8.3 方案 B — EfficientNet-B4 分类器

- **状态**：已完成
- **模型**：EfficientNet-B4（timm 预训练，ImageNet）
- **任务**：7 类疾病分类（正常、类风湿、骨关节炎、痛风、脊柱关节炎、损伤、滑膜囊肿）
- **配置**：384×384, batch=8, 100 epochs, lr=1e-4, AdamW + CosineAnnealing
- **类别不平衡处理**：WeightedRandomSampler + 加权 CrossEntropyLoss
- **数据增强**：RandomHorizontalFlip, ColorJitter, RandomAffine
- **GPU 显存**：~6.8G / 10G（batch=8；batch=16 会占满 9.9G 导致卡机）
- **输出**：`runs/classifier/best.pth`

#### 训练过程

### 8.5 方案 C — MedSAM 微调

- **状态**：已完成
- **模型**：MedSAM（ViT-B，medsam_vit_b.pth 预训练，358MB）
- **任务**：bbox prompt 分割 + 7 类疾病分类联合训练
- **架构**：冻结 image encoder → prompt encoder + mask decoder（可训练）+ 分类头（可训练）
- **配置**：1024×1024 输入, batch=8, 50 epochs, AdamW(decoder lr=1e-5, cls lr=1e-4) + CosineAnnealing
- **损失**：Dice + BCE（分割, λ=1.0）+ CrossEntropy（分类, λ=0.5）
- **预训练权重下载**：从 HuggingFace 镜像 `hf-mirror.com/SansuiHan/medical_models` 获取
- **输出**：`runs/medsam/best.pth`

#### 性能优化

原始脚本直接训练（encoder 未冻结）导致 ViT-B 在 1024×1024 上注意力矩阵巨大（4096 tokens × 12 layers），
反向传播需存储激活张量超出 10GB VRAM，导致内存交换至系统 RAM，速度仅 ~60s/iteration（单 epoch 需 48 小时）。

优化方案：
1. **冻结 encoder + `torch.no_grad()`**：不存储激活张量，显存从 ~18GB 降到 ~1GB
2. **预计算 image embeddings**：5847 个样本共享 2345 张图片，一次性编码后缓存到磁盘（float16，~5GB）
3. 训练时只加载 embedding（256×64×64）+ mask + bbox，跳过 encoder 前向传播

优化后速度提升 ~150 倍：~2.5 it/s，单 epoch ~5 分钟，50 epochs 共 ~3 小时。

**Best Val Dice: 0.8331**（Epoch 34）

#### 分析

1. **分割效果好**：Dice 0.833，给定 bbox prompt 后掩码质量高，快速收敛（Epoch 20 已达 0.832）
2. **分类效果一般**：47.7%，因 encoder 冻结后分类头只能从固定特征学习，特征判别力有限
3. **SAM 批量推理问题**：mask decoder 不支持批量 box prompt，需逐图处理（for-loop）
4. **Val Acc 前 11 epoch 不变**：38.45% 是多数类基线，说明分类头初始阶段未学到有效特征

## 十、技术要点备忘

- **编码问题**：Windows 终端输出中文需设置 `PYTHONIOENCODING=utf-8` 或 `sys.stdout.reconfigure(encoding='utf-8')`
- **患者级拆分**：防止同一患者数据出现在 train 和 test 中导致数据泄漏
- **类别不平衡**：滑膜囊肿仅 36 个标注 vs 类风湿 1006 个，需使用 WeightedRandomSampler 或 Focal Loss
- **MedSAM 输入**：使用 [0, 1] 归一化，不用 ImageNet mean/std
- **nnU-Net**：train + val 合并为 Tr（框架自行做交叉验证），test 作为 Ts
- **GPU 显存限制**：RTX 3080 10GB；YOLO imgsz=1280(9G)、EfficientNet batch=16(9.9G) 都会占满显存导致 Windows 桌面卡死，需降低参数；MedSAM ViT-B 1024×1024 反向传播需 ~18GB，必须冻结 encoder
- **MedSAM 预计算优化**：冻结 encoder + 预计算 embeddings 缓存到磁盘，训练加速 ~150 倍
- **HuggingFace 下载**：国内访问 HuggingFace 需设置 `HF_ENDPOINT=https://hf-mirror.com`
- **Python 3.13 兼容性**：PyTorch 需使用 cu124 索引安装
